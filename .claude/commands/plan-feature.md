Use `INSTRUCTIONS.md` §17 and §19 to plan this feature: $ARGUMENTS.

Produce, in this order and with explicit headings:

1. **Understanding** — restate the objective in one paragraph.
2. **Impact** — which modules, files, and contracts are affected.
3. **Minimum viable scope** — the smallest slice that delivers value.
4. **Risks** — concurrency, consistency, external-failure, regression.
5. **Incremental plan** — ordered steps, each independently testable.
6. **Validation** — how each step will be verified (tests, manual, logs).

Do not start implementing until the plan is approved.
